# Token 成本 —— cairn vs 重型工作流框架

cairn 的核心赌注是：**在检索时付费，不在每次会话付费。** 这篇用可测量的真实数字坐实它，并告诉你怎么用 `cairn-cost.sh` 量自己的。

> "重型工作流框架"那一列是一种**成本模型**，用行为描述——就是[尸检](philosophy.md)针对的那一类。不点名任何工具做稻草人；重点是"每次会话付费 vs 每次检索付费"，不是排行榜。

## 一句话版

| | cairn | 重型工作流框架 |
|---|---|---|
| **每次会话·常驻** | **~350 tok** —— 5 行前言 + 你的在途 🚧/⏸ 面板 | **~5000–7000 tok** —— 每次会话注入一份静态规范 |
| **真正干活时** | SKILL 触发时载一次（~3.5–4.5k tok）+ 一条 grep + 读一个文件 | 已经付过了，不管你用不用 |
| **随时间增长的量** | 在途任务数（有界——收尾即归零） | 框架注入的那一整片 |
| **能不能量** | `cairn-cost.sh`（一条命令、零依赖） | — |

重型框架那个数来自一次真实审计：某项目**每次会话注入 ~22 KB 工作流文档**（~5–7k tok），描述的却是一个没人走的流程。详见[尸检](philosophy.md)。

## 拆开看（实测）

### 常驻税 —— 每次会话都付，不管你碰不碰 cairn
- **固定前言**：~100–130 tok（常量，那 5 行 cairn 说明）。
- **你的在途 🚧/⏸ 面板**：几个任务约 ~180–220 tok。这是**信号**——你手头的活、你正想被提示的东西——不是开销。它也是**唯一**会随时间涨的部分。
- **≈ 合计 ~350 tok**，其中大半是你真实的任务状态。

### 按需 —— 只有真做 cairn 动作才付
- **SKILL 触发时载入**：~3.5–4.5k tok，**每个用到 cairn 的会话一次**——不是每次会话。极少用到的"安装/排障"内容已拆进按需附页，所以常见操作（起任务/收尾/查历史/SOP）**少载 ~800–900 tok**。
- **一条历史 grep**：命中 INDEX 几行，~100–300 tok。
- **读一个任务总结/进度**：~500–1500 tok，且只读你命中的那一个文件——从不整库加载。

### 和重型框架比
那种框架的 ~22 KB（~5–7k tok）是**每次会话**预先付掉的，不管这次是改一行还是跨周迁移——而尸检发现其中大半描述的是没人走的流程。cairn 的常驻成本大约是它的 **1/15–1/20**，而且载的是你的在途任务（信号），不是一份静态文档（噪声）。

## 为什么 cairn 这么省（设计使然）
- **按需检索、零注入**（原则 5）：默认每次会话的常驻面极小；历史只在某个任务真需要时才读。
- **索引 + 详情**（原则 4）：检索 = grep 一个小索引，再读**一个**文件，不是整批加载。
- **约定优先于机器**（原则 6）：没有运行时、没有要保温的向量库、没有要同步的状态文件——所以没有任何东西会悄悄养大每次会话的税。

## 量你自己的
```bash
./.cairn/scripts/cairn-cost.sh            # 常驻足迹 + 按需体积，含面板膨胀告警
./.cairn/scripts/cairn-cost.sh --tiktoken # 装了 python + tiktoken 就出精确值
```
它报出常驻税（固定前言 + 在途面板）、按需体积（INDEX、SKILL），并在在途面板超预算（默认 1500 tok）时告警——提醒你收几个尾，一收就归零。

## 方法与诚实
- 不加 `--tiktoken` 时 token 数是**估算**：中文 ~0.6 tok/字、拉丁/符号 ~0.28 tok/字，贴近现代 o200k / Claude 分词；要精确值用真 tokenizer。
- **22 KB / 5–7k** 是*一个实测项目*的值，不是通用常量——有的重型框架注入更少、有的更多。要记的是**成本模型**（每次会话付 vs 每次检索付），不是某个跑分。
- 这里每个数字都来自本仓库里你能亲手量的产物。没有编造的 benchmark。
