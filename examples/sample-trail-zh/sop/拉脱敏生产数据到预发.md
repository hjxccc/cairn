# SOP —— 拉脱敏生产数据到预发

## 何时用
bug 只在真实数据形态下才复现（并发、编码、金额边界）。

## 前置检查
1. `./scripts/预发健康检查.sh` 返回 OK
2. 你在生产**只读副本**上有 `analyst` 角色——**绝不碰主库**
3. 脱敏工具版本 ≥ 2.3（`anonymize --version`）——旧版会漏掉邮箱域名

## 步骤
1. `export-slice --tables orders,payments,refunds --days 7 --out /tmp/slice.sql.gz`
   预期：约 200–400MB，退出码 0
2. `anonymize --profile pii-strict /tmp/slice.sql.gz`
   预期：报告列出 0 条残留 PII 列
3. `staging-load /tmp/slice.anon.sql.gz --wipe-first`
   预期：打印的行数与导出报告一致 ±0

## 验证
`spot-check.sh --sample 20`——姓名/邮箱是合成的，金额与时间戳保留原样。

## 失败处置 / 回滚
- 步骤 3 部分加载 → 加 `--wipe-first` 重跑（幂等）
- 脱敏报告仍报残留 PII → **停**，不要加载；上报安全渠道
